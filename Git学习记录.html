                              一、Git使用方法

                              1.准备与本地库初始化

                              1)空间准备

                              新建文件夹，准备一个demo文件夹

                               

                              2)基础设置

                              需要进行用户设置，并不影响gitee，只是表明是哪个用户进行的操作

                              • 基本代码

                               

                              3)初始化本地库

                               

                              在文件夹中右键

                              打开 Git Bash Here

                              • 输入 git init -->初始化本地库

                              此时产生一个.git 的文件夹

                              文件夹中的文件不能轻易改变

                              • 使用 “ll” 可以查看目录
                              • 使用"ll -a" 可以查看隐藏目录

                              • 使用"cd 文件名"可以打开文件夹
                              • 使用 "cd .."返回上一级文件夹

                               

                              • git status --> 查看本地库状态

                              on branch status --> 指的是所在分支，分支为 master

                              No commits yet --> 当前未提交过

                              nothing to commit (create/copy files and use "git add" to track) --> 未创建任何文件

                              2.对本地库的操作

                              1)创建一个项目

                              vim 文件名.后缀 --> 创建一个项目

                              • 例：vim hello.txt --> 创建一个名称为hello的文本文档

                              在vim中，先使用Esc退出编辑状态

                              • “yy”是复制；”p“是粘贴
                              • 使用 “：wq” --> 退出保存编辑

                               

                              使用代码检查最后一行的内容

                              2)查看本地库状态

                              此时再次检查本地库状态

                              On branch master --> 当前所处分支，仍为master

                              No commits yet --> 当前未提交过

                              Untracked files: --> 未被追踪的文件 (use "git add ..." to include in what will be committed) hello.txt -->为红色，表示文件虽然存在，但是处于工作区，并未被追踪过

                              nothing added to commit but untracked files present (use "git add" to track)

                              没有添加任何东西，但是已经存在一个未被追踪的文件，可以使用 git add 去追踪🖊

                               

                              3)添加文件到暂存区

                              • git add 项目名称.后缀 --> 添加项目到暂存区

                              warning: LF will be replaced by CRLF in hello.txt.

                              git默认帮助转换了“LF”和“CRLF”两种换行符🖊

                              The file will have its original line endings in your working directory

                              该文件的原始行尾将位于工作目录中🖊

                               

                              再次检查本地库状态

                              On branch master --> 当前所处分支，仍为master

                              No commits yet --> 当前未提交过

                              Changes to be committed:

                              • (use "git rm --cached ..." to unstage) --> 使用此代码进行删除 new file: hello.txt -->为绿色，此文件已被追踪

                              ps: 此时文件只是存在于暂存区中

                              暂存区中的文件可以被删除🖊

                               

                              4)从暂存区删除文件

                              • 使用代码 git rm --cached <file> 进行删除

                              此时删除仅为删除暂存区中的文件，工作区仍存在此文件🖊

                               

                              • 使用“ll”进行验证

                              发现工作区中仍然存在hello.txt文件🖊

                               

                              • 再次使用 git status 进行查看

                              缓存区中的hello.txt文件被删除，需要重新追踪🖊

                               

                              5)提交本地库

                              基本语法：

                              • git commit -m "日志信息" 文件名

                              • git commit -m "first commit" hello.txt

                              warning: LF will be replaced by CRLF in hello.txt.

                              git默认帮助转换了“LF”和“CRLF”两种换行符[master (root-commit) eaf55cb] hello.txt🖊

                              The file will hace its original line endings in your working directory

                              [master (root-commit) 965c6a1] first commit 965c6a1 --> 7位字符码（版本号）

                              在master分支上有一个文件被提交🖊

                              1 file changed, 16 insertions(+)

                              1个文件被改变，16行代码被插入🖊

                              create mode 100644 hello.txt

                               

                              再次查看本地库

                              On branch master --> 在分支master上 nothing to commit, working tree clean

                              已经提交过了，并且在提交之后没有更改过，树是干净的🖊

                               

                              查看版本信息

                              • git reflog

                              eaf55cb (HEAD -> master) HEAD@{0}: commit (initial): hello.txt

                              (HEAD -> master) --> 指针是指向master的

                               

                              • git log --> 查看详细日志

                              可以看到是谁提交的版本以及提交版本的具体时间🖊

                              eaf55cbecfd88ce64140d09209083884756cd511 --> 完整版的版本号

                              6)修改文件

                              • 利用 "cat 文件名"查看文件内容

                               

                              • 利用“vim 文件名”进入文件详情

                              此时可以进行代码修改🖊

                               

                              此时再次查看本地库状态

                              modified: hello.txt -->提示代码已经被修改，并且红色意味着文件还未被添加到暂存区

                               

                              此时需要对文件进行追踪

                              • 利用代码 "git add 文件名" 进行追踪

                               

                              再次进行本地库查询

                              此时文件已经被添加到暂存区中🖊

                               

                              这时进行提交处理

                              • git commit -m "second commit" hello.txt

                              1 file changed, 1 insertion(+), 1 deletion(-)

                              1个文件被修改，1行新增，1行删除（先删除被修改行，再进行添加新的一行）🖊

                               

                              此时进行版本查询

                              • git reflog

                              已经有一个新的版本生成🖊

                              6a2b19b (HEAD -> master)

                              指针指向第二个版本，并且生成一个新的版本号🖊

                               

                              以此类推进行第三次版本提交

                              可以发现第三个版本也已经成功提交，并且生成了一个新的版本号

                              指针总会指向最新的版本🖊

                               

                              进行详细版本号查询

                              可以看到每一个版本的提交时间和是哪个用户提交的🖊

                               

                              此时查看文件，只会看到最后一次修改的文件

                               

                              在工作区中只会有一个文件，不会产生副本

                               

                              7)版本穿梭

                               

                              更改指针指向

                              • git reset --hard 版本号

                               

                              查看版本日志

                              会提示指针已经指向所更改的版本号🖊

                               

                              进入文件查看

                              • cat 文件名

                              fax

                              发现文件版本为刚才更改的版本🖊

                               

                              进入本地文件查看

                               

                              git文件夹中，打开head文件

                               

                              文件中ref: refs/heads/master master即为当前所在分支

                               

                              进入文件夹 refs-->heads-->所在分支

                               

                              会发现其中的版本号就是当前指针所指向的版本

                               

                              再次更改版本为版本1

                              此时文件中的版本号也会随之改变

                              更改版本是从内存中调用不同的版本号进行版本更换，而不是不断复制粘贴新的文件🖊

                               

                              3.git 分支操作

                              服务器运行模式

                              <<<---------------------------------------------------------------------------------------------------------------------------------------

                              在版本控制过程中，同时推进多个任务，为每个任务，我们可以创建每个人物的单独的分支。使用分支意味着程序员可以把自己的工作从开发主线上分离开来，开发自己分支的时候，不会影响主线分支的运行。对于初学者而言，分支可以简单理解为副本，一个分支就是一个单独的副本。（分支底层其实也是指针的引用）

                              分支的好处

                              同时并行推进多个功能开发，提高开发效率

                              各个分支在开发的过程中，如果某一个分支开发失败，不会对其他分支有任何影响。失败的分支删除重新开始即可。

                               

                              分支操作基本代码

                              • git branch -v查看当前所有分支

                               

                              • git branch hot-fix 创建一个名称为hot-fix的新分支

                               

                              查看分支

                              1)更换分支

                              • git checkout hot-fix更换分支

                              当前所处分支已经更换为hot-fix🖊

                               

                              在分支hot_fix上进行文件修改

                              • vim hello.txt 打开文件

                               

                              此时查看本地库状态

                              • git status

                              会提示在hot-fix分支上有一个新的未被追踪的文件🖊

                               

                              此时将文件添加到暂存区

                              • git add hello.txt

                               

                              对文件进行本地库提交

                              • git commit -m "hot-fix first commit" hello.txt

                              此时查看文件，发现文件已经被修改

                              • cat hello.txt

                               

                              进行版本查看，发现指针已经更改指向

                              • git reflog

                              指针指向了hot-fix分支中的新提交的文件🖊

                               

                              2)合并分支(正常合并)

                              进入master分支

                              • git checkout master

                               

                              查看分支下的文件

                              • cat hello.txt

                              文件并未像ho、t-fix中一样被修改🖊

                               

                              合并两个分支

                              • git merge hot-fix 合并hot-fix分支到当前分支上

                               

                              此时查看文件

                              • cat hello.txt

                              发现文件变成已经被修改过的文件了🖊

                               

                              3)合并分支(冲突合并)

                              产生冲突的原因：

                              合并分支时，两个分支在同一个文件的同一个位置有两套完全不同的修改。Git无法替代我们决定使用哪一个。必须人为决定新代码内容。

                               

                              此时进入文件进行修改

                              • vim hello.txt

                               

                              追踪并上传

                              • git add hello.txt

                               

                              • git commit -m "master test" hello.txt

                               

                              检查文件是否被修改

                              • cat hello.txt

                               

                              此时切换分支进行下一步修改

                              • git checkout hot-fix

                               

                              进入文件进行修改

                              • vim hello.txt

                               

                              修改之后进行追踪和上传

                              • git add hello.txt

                               

                              • git commit -m "hot-fix test" hello.txt

                               

                              切换到master分支

                              • git checkout master

                               

                              执行和并指令

                              • git merge hot-fix

                              提示自动合并失败，存在冲突🖊

                               

                              且状态发生改变

                               

                              查看本地库状态

                              • git status

                              会提示有一个项目没有合并成功🖊

                               

                              此时打开文件

                              • vim hello.txt

                               

                               

                              指的是当前分支中的代码🖊

                               

                              指的是冲突分支中需要合并的代码🖊

                               

                              此时删除不需要的代码以及特殊符号，留下需要合并的代码

                              保证代码行数与之前一致，且无特殊符号🖊

                               

                              修改完毕之后输入 Esc --> :wq 保存退出

                               

                              并且需要重新追踪上传

                               

                              但是上传时不能添加文件名，因为两个分支中都存在相同文件

                              如果添加文件名的话，会提示找不到文件🖊

                               

                              进行正确操作

                              • git commit -m "merge test"

                              此时文件成功合并且MERGING 提示也已经消失🖊

                               

                              此时查看文件

                              • cat hello.txt

                              此时两行修改已经成功合并🖊

                              但此时修改的仅为master分支中的内容🖊

                               

                              切换到hot-fix 分支，查看文件内容

                              • git checkout hot-fix
                              • cat hello.txt

                              发现文件并没有被修改🖊

                              说明合并分支只会合并所在分支，被合并分支并不会被合并🖊

                              git合并分支依旧使用的是指针逻辑🖊
